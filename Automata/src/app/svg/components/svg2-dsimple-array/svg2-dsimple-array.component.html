Using SVG, Typescript driven, Simple 2D Array

  • In this case we continue to generate an svg via Typescript and Angular's Renderer2.
  • Now the model has been removed and we render directly from our loop
  • Also Dropped the service reference for building the checkerboard
  • Still used class inheritance but build directky to view. No model!
  • Chrome: 400x400 will give us a delay of 4-5 seconds and a total memory usage of 154MB - an increase of 80MB specifically for the rendering.
  • Chrome: 500x500 will give us a delay of ~15 seconds and a total memory usage of 311MB - an increase of 150MB specifically for the rendering.
  • This is already an increase of 300% on development configuration alone!
  • This drops significantly on production settings - to ~ 64MB in total and 30MB in rendering. There is no huge improvement in rendering performance time as it still takes ~15-20seconds to load.
  • At this point there is no model and optimisations would center around parallel and/or buffered rendering and using wew borkers.
